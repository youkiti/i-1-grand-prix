# 目的

- 審議会資料（YAML: topics, evidence_chunks, spectrum）とパブリックコメント（YAML: topics, evidence_chunks）を比較し、以下の観点で分析を行ってください。
  1. **検証 (Validation)**: パブリックコメントが審議会での議論を支持・補強している点はどこか？
  2. **対立 (Conflict)**: パブリックコメントが審議会での議論に反対、あるいは矛盾する意見を述べている点はどこか？
  3. **発見 (Discovery)**: 審議会では議論されていなかったが、パブリックコメントによって新たに明らかになった視点や論点は何か？

# 出力フォーマット

以下のセクション構成で出力してください。文体は**完全な文章（～です。～ます。）**としてください。
**出力の冒頭に挨拶文や定型文を入れないでください。直接「# 」で始まるセクションから出力してください。**

## ニュートラルな記述ルール（重要）

- **価値判断を含む表現を避ける**: 「強い反対」「大きな支持」「深刻な懸念」などの形容詞は使わないでください。
- **事実ベースで記述する**: 「〜という意見がある」「〜と述べられている」「〜が指摘されている」という形式で、各意見をニュートラルに記載してください。
- **避けるべき表現の例**:

  - ❌ 「強い反対意見が示されている」 → ⭕ 「反対意見が示されている」
  - ❌ 「大きな支持を得ている」 → ⭕ 「支持する意見がある」
  - ❌ 「深刻な懸念が表明されている」 → ⭕ 「懸念が表明されている」

## 最初のセクション: 用語解説（Glossary of Terms）

**このセクションは必須です。** レポートを読む人が高校生でも理解できるよう、本テーマに関する重要な専門用語を5〜10個程度、わかりやすく解説してください。

形式:

```
# 用語解説（Glossary of Terms）

| 用語 | 解説（高校生にもわかる説明） |
|------|------------------------------|
| 用語1 | わかりやすい説明（1〜2文） |
| 用語2 | わかりやすい説明（1〜2文） |
```

用語は入力YAMLの topics や evidence_chunks から抽出し、法律・政治・行政の専門用語、略語、制度名などを優先してください。

## 引用ルール（重要）

- 各論点には必ず **出典タグ** と **原文引用** の両方を明記してください。
- 入力YAMLの `evidence_chunks.verbatim_quote` から原文を引用し、その後に出典タグを付与します。

**引用形式:**

```
> 「原文をそのまま引用」 [出典: ファイル名, p.X] または [パブコメ: コメントID]
```

- 審議会資料からの情報: `[出典: ファイル名, p.X]` （入力YAMLの source_documents と evidence_chunks.page を参照）
- パブリックコメントからの情報: `[パブコメ: コメントID]` （入力YAMLの comment_id を参照）
- 同一論点が両方に関連する場合は、両方の原文引用と出典を併記してください。

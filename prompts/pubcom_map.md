# 前提

あなたは行政官の優秀なアシスタントです。
これから、法案作成に向けたパブリックコメント（意見公募）の分析を行います。
提供された「複数のパブリックコメント（バッチ）」を読み込み、そこに含まれる論点を抽出してください。
各コメントは `=== Comment ID: ... ===` で区切られています。

# 分析対象のパブリックコメント（複数）

{{ referenceDocuments }}

# 分析の主眼

**「{{ focus }}」**に関連する内容にフォーカスしてください。

# 目的

- このコメントが主張している「主要な論点」「懸念点」「新たなアイデア」を抽出すること。
- **重要**: 既存の仮説や議論にとらわれず、このコメントが述べていることを客観的に抽出してください。
- 事務的な詳細（提出様式の不備など）は除外してください。

# 出力フォーマット

**必ずYAML形式**で出力してください。マークダウンのコードブロック（```yaml ... ```）で囲んでください。

## 引用ルール（重要）

- 各論点には必ず **evidence_chunks** として原文を引用してください。
- **verbatim_quote は原文をそのままコピペ**してください。要約ではなく、根拠となる発言・記述そのものを引用します。
- `comment_id` は入力データの `=== Comment ID: ... ===` から取得してください。

## YAMLスキーマ

```yaml
metadata:
  focus: "{{ focus }}"
  batch_comment_ids: ["id1", "id2", ...]  # このバッチで分析したコメントIDリスト
  generated_at: "生成日時"

topics:
  - id: "topic_001"
    title: "論点のタイトル"
    category: "主要論点"  # 主要論点 / 課題・懸念 / その他
    summary: "この論点の要約（なぜ重要か、何が問題か）"
    
    # 論点の幅を可視化するための軸（賛成/反対がある場合）
    spectrum:
      axis: "賛成 ←→ 反対"  # 対立軸（該当しなければ null）
      positions:
        - label: "賛成派"
          description: "賛成理由の概要"
        - label: "反対派"
          description: "反対理由の概要"
      consensus_status: null  # パブコメ段階では通常 null
    
    # 根拠チャンク（原文引用） - 必須
    evidence_chunks:
      - id: "chunk_001"
        comment_id: "12345"  # パブコメID
        verbatim_quote: |
          「原文をそのまま引用。複数行にわたる場合も
          そのまま記載する。」
        position: "賛成派"  # spectrum.positions の label と対応（対立がない場合は null）
        context: "どのような文脈での発言か（任意）"
```

## 出力時の注意

1. **topics は複数抽出**してください。主要論点、課題・懸念、その他をすべて topics 配列に含めます。
2. **evidence_chunks は各 topic に最低1つ**必要です。根拠なき論点は記載しないでください。
3. **spectrum は対立がある場合のみ**記載。対立がなければ `spectrum: null` としてください。
4. **verbatim_quote は省略せず**、根拠として十分な長さの原文を引用してください。
5. YAML として正しくパースできる形式で出力してください。
